Select
Select allows users to select from a single-option menu. Select is a form control for selecting a value from a set of options.
Examples Examples
Basic select
In simple select, option menus positioned under their emitting elements such that the currently selected menu item appears under of the emitting element.

Props Props
To get a customized select, just set ariaLabel / ariaLabelledby / compareWith / disableOptionCentering / disableRipple / disabled / errorStateMatcher / id / multiple / panelClass / placeholder / required / sortComparator / value / openedChange / selectionChange / autofilled / controlType / customTrigger / empty / errorState / focused / ngControl / optionGroups / optionSelectionChanges / options / overlayDir / panel / panelOpen / selected / stateChanges / trigger / triggerValue / source / value
property description type default
ariaLabel Aria label of the select. If not specified, the placeholder will be used as label. string
ariaLabelledby Input that can be used to specify the aria-labelledby attribute. string
compareWith Function to compare the option values with the selected values. The first argument is a value from an option. The second is a value from the selection. A boolean should be returned. (o1: any, o2: any) => boolean
disableOptionCentering Whether to center the active option over the trigger. boolean
disableRipple Whether ripples are disabled. boolean
disabled Whether the component is disabled. boolean
errorStateMatcher Object used to control when error messages are shown. ErrorStateMatcher
id Unique id of the element. string
multiple Whether the user should be allowed to select multiple options. boolean
panelClass Classes to be passed to the select panel. Supports the same syntax as ngClass. string | string[] | Set | { [key: string]: any; }
placeholder Placeholder to be shown if no value has been selected. string
required Whether the component is required. boolean
sortComparator Function used to sort the values in a select in multiple mode. Follows the same logic as Array.prototype.sort. (a: MatOption, b: MatOption, options: MatOption[]) => number
value Value of the select control. any
openedChange Event emitted when the select panel has been toggled. EventEmitter
selectionChange Event emitted when the selected value has been changed by the user. EventEmitter
autofilled Whether the input is currently in an autofilled state. If property is not present on the control it is assumed to be false. boolean
controlType A name for this control that can be used by mat-form-field. 'mat-select'
customTrigger User-supplied override of the trigger element. MatSelectTrigger
empty Whether the select has a value. boolean
errorState Whether the control is in an error state. boolean
focused Whether the select is focused. boolean
ngControl NgControl
optionGroups All of the defined groups of options. QueryList
optionSelectionChanges Combined stream of all of the child options' change events. Observable
options All of the defined select options. QueryList
overlayDir Overlay pane containing the options. CdkConnectedOverlay
panel Panel containing the select options. ElementRef
panelOpen Whether or not the overlay panel is open. boolean
selected The currently selected option. MatOption | MatOption[]
stateChanges Stream that emits whenever the state of the control changes such that the parent MatFormField needs to run change detection. Observable
trigger Trigger that opens the select. ElementRef
triggerValue The value displayed in the trigger. string
source Reference to the select that emitted the change event. MatSelect
value Current value of the select that emitted the event. any
Usage & Content Guidelines Content Guidelines
Select menu should be short, clear, and fully visible. It shouldn’t be truncated.
