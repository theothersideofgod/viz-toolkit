Header & Footer
Header & Footer appear on top & botton all other surfaces in an app, displaying information and actions relating to the current screen.
Examples Examples
Header & Footer are a special kind of toolbar that’s used for branding, navigation, search, and actions.
Props Props
To get a customized header & footer, just set title/userName/brandLink/svgIcon/type/leftBrands/rightBrands
property description type default
title Title for the header. string
userName UserName for the navbar-right. string
brandLink Link for the brand. string
svgIcon Url for the svgIcon. string
type Type of viz-header. string primary
leftBrands List of links on left part of viz-footer. array [ ]
rightBrands List of links on right part of viz-footer. array [ ]
rightBtnClick The callback function that is triggered when right button is pressed. function(e)
hamburgerClick The callback function that is triggered when hamburger button is pressed. function(e)
Best Practices Best Practices
Header and Footer should:
  • Persistent: Header and Footer appear at the top and bottom of each screen in an app, and can disappear upon scroll.
  • Guiding: Header and Footer provide a reliable way to guide users through an app.
  • Consistent: Header and Footer have a consistent position and content to increase familiarity.
  • Immediate: Header and Footer can be accessed from any scroll position.
Usage & Content Guidelines Content Guidelines
If title text is long, use a prominent app bar and wrap the title to two lines. Don’t truncate text, hiding the full title may cause misunderstanding. Don’t shrink text to fit on a single line.
