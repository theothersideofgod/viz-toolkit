Form Field
Text fields let users enter and edit text.
Examples Examples
Text fields allow users to enter text into a UI. They typically appear in forms and dialogs.
Hint text conveys additional guidance about the input field, such as how it will be used.
Area form fields are taller than default form fields and wrap overflow text onto a new line. They are a fixed height and scroll vertically when the cursor reaches the bottom of the field.
When text input isn’t accepted, the field turns to red.
A disabled form field is unusable and un-clickable and are rendered in gray by default in browsers.

Best Practice Best Practices
Form field should:
  • Be clearly labeled so it’s obvious to the users what they should enter into the field
  • Only ask for information that’s really needed
  • Validate input as soon as user has finished interacting with a field
