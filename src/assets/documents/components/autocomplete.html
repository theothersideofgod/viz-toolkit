Props Props
To get a customized autocomplete, just set autoActiveFirstOption / classList / disableRipple / displayWith / panelWidth / closed / opened / optionSelected / id / isOpen / panel / showPanel
property description type default
autoActiveFirstOption Whether the first option should be highlighted when the autocomplete panel is opened. Can be configured globally through the MAT_AUTOCOMPLETE_DEFAULT_OPTIONS token. boolean
classList Takes classes set on the host mat-autocomplete element and applies them to the panel inside the overlay container to allow for easy styling. string
disableRipple Whether ripples are disabled. boolean
displayWith Function that maps an option's control value to its display value in the trigger. ((value: any) => string) | null
panelWidth Specify the width of the autocomplete panel. Can be any CSS sizing value, otherwise it will match the width of its host. string | number
closed Event that is emitted when the autocomplete panel is closed. EventEmitter
opened Event that is emitted when the autocomplete panel is opened. EventEmitter
optionSelected Event that is emitted whenever an option from the list is selected. EventEmitter
id Unique ID to be used by autocomplete trigger's "aria-owns" property. string
isOpen Whether the autocomplete panel is open. boolean
panel Element for the panel containing the autocomplete options. ElementRef
showPanel Whether the autocomplete panel should be visible, depending on option length. boolean
Best Practice Best Practices
Form field should:
  • Be clearly labeled so it’s obvious to the users what they should enter into the field
  • Only ask for information that’s really needed
  • Validate input as soon as user has finished interacting with a field
