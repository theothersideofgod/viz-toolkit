Datepicker
The Datepicker component allows the user to enter a date or date range either through text input or by choosing from the calendar.
Examples Examples
Basic Datepicker
Use basic datepicker to let user enter a date.
Datepicker Start Date
It can be set to month, year, or multi-year; by default it will open to month view. The month, year, or range of years that the calendar opens to is determined by first checking if any date is currently selected, if so it will open to the month or year containing that date. Otherwise it will open to the month or year containing today's date.
Datepicker Selected Value
Datepicker with Min & Max Validation
There are three properties that add date validation to the datepicker input. The first two are the min and max properties. In addition to enforcing validation on the input, these properties will disable all dates on the calendar popup before or after the respective values and prevent the user from advancing the calendar past the month or year (depending on current view) containing the min or max date.
Datepicker with Filter Validation
this will also disable the dates on the calendar that are invalid. However, one important difference between using this datepicker vs using Datepicker with Min & Max Validation is that filtering out all dates before or after a certain point, will not prevent the user from advancing the calendar past that point.
Datepicker Emulating a Year and Month Picker
When a year or a month is selected in multi-year and year views respectively, the selected year and selected month outputs emit a normalized date representing the chosen year or month. By "normalized" we mean that the dates representing years will have their month set to January and their day set to the 1st. Dates representing months will have their day set to the 1st of the month.

Best Practice Best Practices
The Datepicker component should:
  • Allow user to input a date or date range.
  • Be inputed through text input or by choosing date from calendar.
  • Always display the current date by default.
Usage & Content Guidelines Content Guidelines
Always disable the dates that the user should not be able interact with.
