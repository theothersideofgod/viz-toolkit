To get a customized datepicker, just set calendarHeaderComponent / color / dateClass / disabled / opened / panelClass / startAt / startView / touchUi / closedStream / monthSelected / openedStream / yearSelected / id
property description type default
calendarHeaderComponent An input indicating the type of the custom header component for the calendar, if set. ComponentType
color Color palette to use on the datepicker's calendar. ThemePalette
dateClass Function that can be used to add custom CSS classes to dates. (date: D) => MatCalendarCellCssClasses
disabled Whether the datepicker pop-up should be disabled. boolean
opened Whether the calendar is open. boolean
panelClass Classes to be passed to the date picker panel. Supports the same syntax as ngClass. string | string[]
startAt The date to open the calendar to initially. D | null
startView The view that the calendar should start in. 'month' | 'year' | 'multi-year'
touchUi Whether the calendar UI is in touch mode. In touch mode the calendar opens in a dialog rather than a popup and elements have more padding to allow for bigger touch targets. boolean
closedStream Emits when the datepicker has been closed. EventEmitter
monthSelected Emits selected month in year view. This doesn't imply a change on the selected date. EventEmitter
openedStream Emits when the datepicker has been opened. EventEmitter
yearSelected Emits selected year in multiyear view. This doesn't imply a change on the selected date. EventEmitter
id The id for the datepicker calendar. string
